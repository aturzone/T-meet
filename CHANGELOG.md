## [Unreleased]

### Added
- Phase 03 — Rooms & Auth: `meet_core::auth` — PASETO v4 local tokens (admin sealed with the per-server 32-byte secret; room tokens sealed with a per-room 32-byte secret, encoding `pid` + `dn` claims). Hard TTL caps (admin 24 h, room 12 h) enforced inside the issuer. `meet_core::auth::password` provides argon2id hashing (m=64 MiB, t=3, p=1) with `subtle`-wrapped verification. `meet_core::auth::room_secret` generates and seals per-room secrets with the admin-passphrase-derived key. `meet_core::ids` mints opaque 22-char base64url room/participant IDs and 26-char base32 server-generated passwords (128 bits each). `meet-server::admin_secret` adds a `data/admin.bin` sealed blob; `init` mints the admin token (printed ONCE in the first-boot banner) and `load_or_rotate` returns the admin secret + at-rest key for the running server. New routes: `POST /admin/rooms`, `GET /admin/rooms`, `GET /admin/rooms/:id`, `DELETE /admin/rooms/:id` (all behind `middleware::admin_auth` bearer check), and the public `POST /r/:id/join` returning `{join_token, ws_url, ice_servers, participant_id}`. New `middleware::rate_limit` (5 / minute / `(ip, room_id)` for join, with `Retry-After`). Generic `401` for both wrong password and unknown room, audit-log entries for every admin action and every join attempt (success and failure). Three new integration tests: admin CRUD round-trip, join-flow happy/sad/rate-limit, and admin-token verification across an `init` → `serve` boundary.
- Phase 02 — HTTP Server & Storage: `meet_core::db` — `Db` newtype over `sqlx::SqlitePool` (WAL + NORMAL synchronous + foreign_keys ON, file chmod 0600 on create), embedded migration set `migrations/0001_init.sql` covering `rooms` (with cascading delete to participants), `participants`, `audit_log`, plus typed CRUD modules for each. `meet-server` gains an `app::build_app` router + `AppState` (Arc<Db>, Arc<DataPaths>), middleware stack (`request_id` UUID v4 + `x-request-id` echo, `access_log` info-level with method/path/status/latency/request_id and no IP, `security_headers` baseline CSP + HSTS + nosniff + Referrer-Policy + Permissions-Policy + COOP/CORP, `body_limit` 1 MiB JSON), `routes::assets` (rust-embed + SPA-friendly fallback with reserved-prefix passthrough, hashed-asset Cache-Control: immutable, no-store on index.html, weak ETag), and a plain-HTTP listener that 301s everything to HTTPS while still serving `/ca.crt` as a locked-down-device escape hatch. Two new integration tests: `meet.db` lifecycle + chmod 0600, and end-to-end coverage of the HTTPS surface, security headers, SPA fallback, reserved-prefix 404, HTTP→HTTPS redirect, and the plain-HTTP `/ca.crt` exception.
- Phase 01 — TLS & Crypto: `meet_core::crypto` submodules — argon2id passphrase KDF (m=64 MiB, t=3, p=1), `XChaCha20Poly1305` AEAD seal/open with random per-call nonce and AAD, `rcgen`-based CA + leaf issuance with IP/DNS SAN entries, on-disk CA blob format (magic + version + salt + sealed plaintext), pure rotation policy with injectable clock (reissue at age ≥ 60d). `meet-server` gains `init`/`serve` commands: `init` reads the passphrase from `MEET_ADMIN_PASSPHRASE` or `rpassword`, generates CA + first leaf, persists `data/ca.bin` (0600), `data/ca.crt`, `data/leaf.pem`, `data/leaf.key` (0600), and prints the leaf SHA-256 fingerprint once. `serve` loads the encrypted CA, auto-rotates the leaf when due, builds a rustls (ring) TLS 1.3 server config, and serves `GET /ca.crt` + `GET /healthz` over HTTPS via `axum-server`. Three integration tests: file layout + permissions, second-init refusal, and end-to-end TLS handshake against `/ca.crt` with the locally-generated CA.
- Phase 00 — Foundation: Cargo workspace (`meet-core`, `meet-server`, `meet-sfu`) with `#![forbid(unsafe_code)]`, config schema with TOML loader + validators, `tracing` + `EnvFilter` logging skeleton (pretty / JSON formats), `meet-server` CLI stubs for `init`/`serve`/`--version`/`--help`. Vite + React 18 + TypeScript strict + Tailwind frontend skeleton with one Vitest smoke test. `justfile` wired for `build`/`check`/`fmt`/`lint`/`test`/`clean`; `scripts/check.sh` mirrors CI gates locally. `rust-toolchain.toml` pins to stable; `clippy.toml` MSRV synced to `1.83`.
- Stage A scaffolding: full repository skeleton, planning documents for phases 00–11, MCP / CI configuration, tooling configs, open-source meta files.
